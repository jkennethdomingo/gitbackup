Git Workflow Tutorial

Step 1: Initialize a Git Repository

git init

This will create a new Git repository in the current directory.

Step 2: Staging Changes

git add .

This command stages all the modified files in the current directory. Alternatively, you can specify individual files to stage:

git add filename

Step 3: Check the Status of Your Repository

git status

You can see which files are staged and unstaged by running the status command.

Step 4: Commit Your Changes

git commit -m "Your commit message here"

This creates a new snapshot of your project with the current changes.

Step 5: View Commit History

git log

This displays a list of previous commits, including the commit message, hash, and author details.

Pushing to a Remote Repository (GitHub)

After making commits locally, you can push them to a remote repository, such as GitHub. Follow these steps:

Step 1: Link Your Repository to GitHub

git remote add origin https://github.com/yourusername/yourrepository.git

Step 2: Push Your Changes

git push -u origin master

This will push your changes to the master branch on GitHub.
